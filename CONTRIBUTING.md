# Contributing

This is primarily a personal portfolio and case-study repository. External
contributions are not expected, but the working standards below apply to anyone
(human or AI agent) making changes.

## Before You Start

1. Read [`AGENTS.md`](AGENTS.md) — the authoritative project instructions.
2. Read [`SECURITY.md`](SECURITY.md) — the publication and sanitization policy.
3. Confirm `source-private/` is git-ignored: `git check-ignore source-private/`.

## Golden Rules

- **Never commit anything from `source-private/`** or any raw production data.
- **Never invent a metric.** Every count must trace to a source file.
- **Distinguish verified facts from interpretation** in everything you write.
- **Do not commit to `main`.** Branch, then open a pull request.

## Branching

- Branch names describe the work: `scaffold/…`, `feat/…`, `docs/…`, `fix/…`.
- Keep commits small and logical; write clear commit messages.

## Commit Message Style

Use conventional-commit prefixes:

- `feat:` new public artifact or capability
- `docs:` documentation additions or edits
- `chore:` governance, tooling, or housekeeping
- `fix:` correction to a published artifact or script
- `test:` test additions or changes

## Adding or Regenerating Sanitized Outputs

- Public data derivatives are generated by scripts in `scripts/`.
- Scripts read from `source-private/` (local only) and write only sanitized
  outputs to tracked paths.
- After regenerating, re-run the privacy checks described in `SECURITY.md`
  before committing.

## Sample Data

- Sample records must be clearly fictional and carry an explicit marker
  (e.g., a top-level `"sample_data": true` field in JSON).
- Sample data illustrates structure. It must never resemble real records.

## Definition of Done

- [ ] No confidential or personal data is present.
- [ ] All stated metrics trace to a source and are labeled by evidence tier.
- [ ] All internal repository links resolve.
- [ ] Unfinished work is labeled as unfinished.
- [ ] Changes are on a branch with a pull request, not on `main`.

# Documentation Roadmap

This directory explains the business problem, system boundaries, design decisions, governance model, and implementation constraints behind the ecosystem.

## Planned Documents

1. `01-overview.md` — organization context, project scope, and author responsibilities
2. `02-business-problem.md` — operational fragmentation and user needs
3. `03-system-architecture.md` — six-area environment and application boundaries
4. `04-data-model.md` — custom and extended Dataverse architecture
5. `05-grants-and-outcomes.md` — funder requirements, KPIs, measures, compliance, and reporting
6. `06-development-finance.md` — donor operations, transactions, deposits, budgets, and reconciliation
7. `07-service-navigation.md` — resource taxonomy and goal-pathway system
8. `08-automation.md` — Power Automate, document processing, and exception handling
9. `09-governance.md` — privacy, permissions, documentation, testing, and maintenance
10. `10-limitations-and-next-steps.md` — known constraints and future improvements

## Writing Standard

Each document should answer:

- What problem existed?
- Who used the system?
- What did Taylor design or build?
- Why was the selected approach appropriate?
- What tradeoffs or constraints shaped the implementation?
- What measurable or observable result followed?
- Which sanitized artifact supports the claim?
